## Section 43: Team Guidelines

### For Code Reviewers

**Review Checklist**:
- [ ] API responses use `apiSuccess()` / `apiError()` helpers
- [ ] Error messages are user-friendly (no stack traces)
- [ ] Server logging uses `createLogger()`, not `console.*`
- [ ] No new `as any` type casts without justification
- [ ] Documentation matches implementation
- [ ] Tests cover happy path and error scenarios

### For Frontend Developers

- Use Sentry (`captureError()`) for client-side errors
- Import from absolute paths (`@/lib/...`, not `../../../`)
- Follow naming conventions from `CODE_STANDARDS.md`
- Add JSDoc comments to exported functions/components

### For Backend/API Developers

- Use `responses.ts` helpers for all API responses
- Use `createLogger()` for logging
- Validate input with Zod or TypeScript
- Add request tracing with requestId
- Test error paths (400, 401, 404, 500)
