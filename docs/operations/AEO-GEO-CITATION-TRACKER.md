> Purpose: Increase Resurgo visibility in AI-generated answers by systematically tracking citations, fixing gaps, and running weekly experiments.
>
> Owner: Marketing lead
>
> Cadence: Weekly (Friday)
>
> Last updated: 2026-04-20

---

## 1) North Star + Success Criteria

### North star KPI

- **Citation Rate** = `Resurgo citations / total checks`

### Secondary KPIs

- **Primary Citation Rate** = `Primary placements / total checks`
- **Engine Coverage** = `# engines citing Resurgo at least once / 4`
- **Query Coverage** = `# queries where Resurgo is cited at least once / 20`
- **Competitive Share of Voice (SOV)** = `Resurgo citations / (Resurgo + top 3 competitor citations)`

### Targets

- Weekly Citation Rate: **>= 30%**
- Weekly Primary Citation Rate: **>= 12%**
- Engine Coverage: **>= 3/4**
- Query Coverage: **>= 10/20**

## 6) Action Rules (non-negotiable)

### If any cluster Citation Rate < 20%

1. Refresh related page(s) with a direct-answer intro (40–80 words).
2. Add/refresh FAQ block with 3–5 high-intent Q&A pairs.
3. Add 1 concrete proof point (data, screenshot, or benchmark).
4. Add 2 internal links from relevant pages with intent-matching anchor text.
5. Re-submit updated URL(s) in Search Console the same day.

### If competitor dominates a query (appears primary 2+ engines)

1. Build/refresh dedicated comparison page.
2. Add explicit `Why choose Resurgo` section with capability-level contrast.
3. Add clear CTA + social proof + product evidence above fold.

### If volatility = High across 2+ engines

1. Mark query as `unstable`.
2. Prioritize foundational pages (category, use-case, comparison).
3. Re-check next week before large rewrite.

## 7) Weekly Operating Runbook

Run this sequence every Friday without skipping:

1. Execute all 80 checks (20 queries x 4 engines).
2. Fill the tracking table live.
3. Calculate weekly scorecard immediately.
4. Flag underperforming clusters + unstable queries.
5. Create/assign content actions same day.
6. Submit refreshed URLs in Search Console.
7. Log one falsifiable hypothesis for next week.

### Weekly owner checklist

- [ ] Query run completed (80 checks)
- [ ] Scorecard completed
- [ ] Underperforming clusters identified
- [ ] Content refresh tasks created and assigned
- [ ] Search Console resubmissions completed
- [ ] Next-week hypothesis logged

## 11) QA Guardrails

- Never log a citation without URL evidence.
- Never compare weeks with changed query text unless explicitly noted.
- Keep one source of truth (this file or linked sheet, not both unsynced).
- Every action item must map to a measurable query/cluster gap.
- Every monthly report must include at least one `stop doing` decision.

---

## 12) Definition of Done (weekly cycle)

Weekly cycle is only complete when:

1. Scorecard is filled,
2. Actions are assigned,
3. Search Console submissions are done,
4. Next-week hypothesis is written.

If any one of these is missing, status = **Incomplete**.
